JaReader 重构目标
===============

这是一个通过日语生肉小说阅读来提高日语水平的阅读器。

核心理念：轻量化、免费、开源、好用。
目标用户：中文用户，TTS / 翻译在大陆网络环境下可用。
技术栈：React Native + Expo SDK 57（TypeScript/React）


1. 跨平台
   Android + iOS，兼容性优先。目前优先实现安卓。

2. 格式支持（渐进）
   txt（内置）+ epub（JSZip 自解析，~50KB）
   导入的书籍复制存储在手机首页目录，方便查找。

3. 主页面极简
   - 标题栏：下沿有单侧阴影边框，左侧是标题Jareader，右侧是设置按钮（图标为三横线）点开后显示悬浮的子页面，包括使用操作（导入方法、支持格式、应用特性、隐私说明，以及整段翻译功能的key导入）
   - 图书列表
   - 右下角 FAB 导入按钮。横竖屏自适应。

4. 阅读界面——单手操作设计
   - 句段居中，四边充足留白
   - 整体自上而下分为四个区域：顶部栏目区、翻译结果卡片区、文本卡片区、底部操作栏区。顶部栏目区位置和尺寸固定、后三者根据内容量和操作模式自适应空间分配比例，但区域间显示内容和操作逻辑隔离，这是交互实现的核心约束。
   - 顶部栏（窄条）：纲目 + 进度（进度条+xx%） + 手模式切换（双手/左手/右手）+横竖屏切换（不接受手机重力感应自动切换），窄条下沿有单侧阴影边框，窄条本身很窄，磨砂玻璃质感，内部控件按钮静息时没有边框，点击时按钮整体阴影。
   - 翻译结果卡片，在文本卡片上方，不遮挡文字，动态大小和显示，只在用户选中内容时触发翻译和显示，文本卡片以上、顶部栏以下区域均为其操作区，点击操作区中卡片以外的区域即可关闭卡片显示，卡片显示切换和内容改变时卡片尺寸和卡片间布局均刷新。
   - 文本卡片：点按=查词，横滑=手动选词 不需要长按，提高流畅。
   - 底部操作栏：按钮卡片：◀/🔊/▶ + 整段翻译 图标不符合风格，应改换极简风格。卡片上方空白区域有滑动翻页功能，上下左右滑动均接收有效。但仅限该区域，其余区域（文本区、翻译区等）均不检测翻页手势，这是实现滑动选词和单击时不用长按且不会误触的核心手段。
   - 单手模式：按钮聚集单半侧——左（上/下）/右（上/下）交错排列防误触，当前的交错过于简单仅仅是将横排改为了竖排。文本卡片也会贴近单侧，为视觉平衡，字典卡片偏向对侧。
   - 阅读区与操作区物理隔离 → 多手势共存无冲突 上面已经说过，且当前该功能没有实现。
   - 横屏文本区宽度加长高度降低，以继续满足以上上下分区方案。

5. 翻页——三通道
   上下滑+ 左右滑+ 按钮点击
   动画：Reanimated 3，300ms，60fps

6. 自动分句——以完整语义段为单位
   正则 + 启发式（0KB）。。！？为主分隔符。引号内不分割。
   精确率 95%+。
   特点（核心）：以句段为单位，而不是传统阅读器的铺满整夜的页为单位为屏幕剩下大量空余空间，得以实现
   在导入时进行该工作，实际操作是将导入文件解析copy为文本格式的内容+插图（如果有）然后存储到应用的目录中。以打开阅读时能快速解析为目标设置格式和方法。
   导入过程耗时较多，有用户选择完文件后先检查格式是否为epub或txt，不是则提醒并返回主页面，是则创建新条目显式显示导入处理中，处理完成则该条目内容显示正式的图书条目内容，失败则显示可能原因并取消该条目。

7. 词典——本地、免费、无版权
   数据：JMdict → SQLite FTS5
   分词：kuromoji 尝试 → 自动降级正则
   点按即查 → 结果框显示读音+释义+词性
   松手即查，无长按等待
   直接内涵到安装包中本地使用，不用用户使用时现场安装。如有更轻量化词典源实现可采纳。

8. 整段翻译——辅助功能、自定义、整个应用唯一需要外部联网功能的地方
   代理架构：App → DeepSeek API-嵌入翻译请求提示词+该段文本->返回文本翻译显示在翻译结果卡片上。
   首次启动自动打开设置子页面引导用户如何自己导入apikey，不导入不影响使用，只是没有整段翻译功能。
   后台调用不跳页，译文存本地建立好翻译-纲目映射，下次翻译直接拉取。
   apikey属于用户隐私，做好保护措施和显式声明。

9. 纲目——圣经式句段导航
   每句编号（章.句）。搜索 + 按章分组 + 点击跳转。书签高亮。分句存储也适用该方案，统一用一种标记方案能减少后续麻烦。

10. TTS 朗读——免费、自动、稳定、手机自带功能
    expo-speech → 自动检测日语语音 → 系统 TTS
    应用初次启动时自动检查
    超时重试 3 次。底部 🔊 按钮阅读当前句段（单手可达）。
    无自动连读

11. 开源、不花钱、利用现有能力
    所有工具链免费/开源。JMdict(CC-BY-SA)。设备内置 TTS。
    DeepSeek 极低价 API。
    字体：内置 Noto Sans JP（开源 ~5MB）


UI 关键约束
- 单手操作，竖屏横屏分别设计
- 顶部切换双手/单手左右手
- 单手时操作区在屏幕下方靠左/右单侧
- 功能按钮不对称，左右手模式镜像翻转
- 按钮交错排列防误触
- TTS 按钮在底部操作区（单手可达）
- 字典结果在操作区内专用区域显示
- 纲目等低频功能在顶部栏
- 插图独占卡片，显式图片时不显示文本。
- 文本统一横排
- 翻译 API：大众了解、连接方便、申请无门槛（DeepSeek 首选）
- 卡片样式轻量极简，使用纯色背景、轻微阴影、无圆角或小圆角。
