
整体风格参考：

* Apple Books 的留白
* Kindle 的阅读沉浸感
* Obsidian 卡片式布局
* iOS Human Interface Guidelines 的轻量化交互

---

# 一、整体布局

整个阅读界面采用**垂直四分区布局**。

```
┌──────────────────────┐
│ 顶部栏               │
├──────────────────────┤
│ 翻译结果区           │
├──────────────────────┤
│ 文本阅读区           │
├──────────────────────┤
│ 底部操作区           │
└──────────────────────┘
```

整个页面不允许自由堆叠组件。

四个区域必须保持逻辑隔离。

除了顶部栏固定高度，其余三个区域根据内容动态调整高度。

但是：

> **区域之间不能发生交互冲突。**

例如：

* 文本区永远只负责阅读与选词。
* 操作区永远只负责翻页和控制。
* 翻译区永远只负责展示翻译。

任何手势都不得跨区域响应。

这是整个交互设计最重要的原则。

---

# 二、顶部栏（固定高度）

顶部栏始终固定在页面顶部。

高度：

约 44~52dp。

采用：

* 半透明磨砂玻璃(Material Blur)
* 极窄高度
* 下沿存在柔和阴影
* 与正文形成视觉分层

顶部栏包含：

左侧：

当前章节名称

中间：

阅读进度

* ProgressBar
* 百分比

右侧：

两个切换按钮：

① 左手 / 双手 / 右手模式

② 横屏 / 竖屏切换

禁止依赖：

* 手机重力自动旋转

必须由用户主动点击切换。

按钮设计：

默认：

* 无边框
* 无背景

点击：

* 卡片浮起
* 阴影
* Material Press Animation

# 三、翻译结果区（动态区域）

位于顶部栏下方。

默认：

高度为0。

即：

没有翻译内容时完全收起。

只有：

用户查词

或者

整句翻译

才展开。

展开后显示：

翻译结果卡片。

翻译区采用：

卡片式布局。

特点：

背景：

米黄色

纸张质感

类似：

学习笔记

每一行存在浅色横线：

像手写笔记本。

卡片圆角：

16dp

阴影：

极弱

强调纸张感。

---

## 卡片内容

支持两种模式：

① 单词解释

例如：

```
食べる

【动词】

吃

食用

进食

词形：

食べます

食べた

食べない
```

② 整句翻译

例如：

```
今天虽然下雨，

但是还是决定出去。
```

两者共享同一展示区域。

不会同时显示多个卡片。

新翻译直接替换旧内容。

切换时：

卡片尺寸重新计算。

布局重新动画。

---

## 空白关闭机制

翻译区除卡片以外全部属于：

Dismiss Area

点击：

立即关闭翻译。

关闭动画：

向上收缩。

文本区自动恢复高度。

# 四、文本阅读区（核心区域）

这是阅读器最重要区域。

整个区域：

只允许存在：

文本卡片。

不允许：

工具按钮

菜单

悬浮球

广告

等元素。

---

## 文本布局

一句话为一个阅读单元。

句子居中。

左右留白充足。

上下留白略大。

阅读视线集中。

字体：

偏日文阅读字体。

支持：

ruby

振假名。

句子之间保持明显分隔。

避免长段连续文本。

---

## 文本交互

这是整个产品最重要交互。

### 点击

点击任意位置：

自动识别最近单词。

立即：

自动选中。

无需长按。

立即查词。

显示：

翻译卡片。

---

### 横向滑动

左右滑动：

进入：

自由选词模式。

允许：

拖动选区。

实时更新：

选中文字。

结束滑动：

自动查词。

无需菜单。

无需复制。

无需二次确认。

---

### 不允许

文本区：

绝不能响应：

上下翻页。

否则：

会影响：

滑动选词。

因此：

文本区所有纵向滑动全部忽略。

---

# 五、底部操作区（翻页控制）

该区域是：

唯一允许翻页的位置。

采用：

按钮卡片 + 大面积空白。

布局：

```
      ◀
 译 🔊
  ▶


```

按钮采用：

Material Card

点击动画：

轻微弹起。

---

## 空白区域

按钮之间、

按钮外侧、

整个底部区域剩余空间，

全部属于：

翻页操作区。

支持：

任意方向滑动：

* 左
* 右
* 上
* 下

全部解释为：

上一句

下一句。

因此：

用户无需精确滑动方向。

只要滑即可翻页。
示意：
│        文本区         │
├──────────────────────┤
│      空白可滑动区     │
├──────────────────────│
│        按钮栏        │
└──────────────────────┘
---

## 为什么这样设计

阅读区：

没有翻页。

操作区：

负责全部翻页。

实现：

阅读操作

与

翻页操作

物理隔离。

这样：

文本区：

可以自由：

点击

拖动

选词

而不会误翻页。

这是整个交互设计的核心。

# 六、单手模式

支持：

* 双手
* 左手
* 右手

三种布局。

按钮自动重排。

例如：

右手模式：

```
   ▶
🔊 译
      ◀

整段翻译
```

左手模式：

镜像排列。

注意：

不是简单竖排。

而是：

交错排列。

保证：

按钮点击区域不会重叠。

降低误触。

---

## 文本位置

单手模式下：

正文轻微向操作手一侧偏移。

例如：

右手：

正文略向右。

左手：

正文略向左。

形成：

拇指自然阅读区。

---

## 翻译卡片

为了视觉平衡：

正文偏左：

翻译卡片略偏右。

正文偏右：

翻译卡片略偏左。

保持：

整体重心居中。

---

# 七、横屏模式

横屏不是简单旋转。

而是重新布局。

保持：

四分区思想。

变化：

顶部：

保持固定高度。

正文：

变宽。

降低高度。

保证：

单句阅读宽度增加。

减少换行。

底部：

保持操作区。

按钮仍位于操作手附近。

---

# 八、动态布局规则

除顶部栏固定高度外：

翻译区、

正文区、

底部区

均采用弹性布局。

布局优先级：

1. 文本阅读区优先保证可读性。
2. 翻译区仅按内容高度展开，不占用超过约 40% 屏幕高度；超出部分内部滚动。
3. 底部操作区保持最小可点击尺寸（满足约 44dp 的触控目标）。
4. 翻译区关闭时，其空间立即归还给文本区。
5. 所有区域切换采用平滑动画（约 200–300ms），避免界面跳变。
